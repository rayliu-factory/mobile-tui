    Constructor: `PropertyInspectorPane(store: Store, getActiveScreenId: () => string | null, theme: CanvasTheme)`

    `update(snapshot: Snapshot): void` — stores snapshot, invalidates cache.

    `render(width: number)`:
    - Look up active screen from `snapshot.spec.screens.find(s => s.id === getActiveScreenId())`
    - If no screen: return `[truncateToWidth("(no screen selected)", width)]`
    - Build 5 field rows from the screen:
      1. `title: <value>` — editable; cursor indicator `>` when cursorRow=0 and focused
      2. `back_behavior: <value or "—">` — editable unless root screen (back_behavior===undefined)
      3. `components: N components` — read-only
      4. `acceptance: <N lines or "(none)">` — editable (shows count when not expanded)
      5. `variants: content ● empty ● loading □ error □` — read-only (use actual variant nullity)
    - If `editingField` active, replace that field row with the `this.editInput.render(fieldWidth)` output
    - Diagnostics: check `snapshot.diagnostics` for severity="error" where path includes active screenId; prepend `⚠ ` to affected field row
    - Error count summary as last line if any errors: `  N error(s)`
    - Apply theme in render(), never in constructor: `this.theme.fg("accent", cursor)` for cursor
    - Every line: `truncateToWidth(line, width)` (Pitfall 1)

    `handleInput(data: string)`:
    - When `editingField` is not null: delegate to `this.editInput.handleInput(data)`; also handle Esc and Enter directly (see D-71)
    - When `editingField` is null: `j`/down → cursorRow++; `k`/up → cursorRow--; Enter → activateField()
    - `activateField()`: for editable fields, set `editingField = fieldKey`, call `editInput.setValue(currentValue)`, set `editInput.focused = this._focused`

    `commitEdit()`:
    - Reads `this.editInput.getValue()`
    - Calls `this.store.apply(commandName, args)` with correct command per field
    - Clears `editingField`

    Focusable implementation:
    - `get focused()` returns `_focused`
    - `set focused(v)` sets `_focused = v`; if `editingField !== null`, also sets `this.editInput.focused = v`

    Test cases to add in canvas-render.test.ts (stub store, mock screen):

    - render(40) with a screen → contains "components:" somewhere
    - render(40) with error diagnostic → at least one line contains "⚠"
    - All lines visibleWidth <= 40

<threat_model>
## Trust Boundaries

| Boundary | Description |
|----------|-------------|
| User keyboard → store.apply | User types text in Input; on Enter, that value flows to store.apply as a command arg |
| Spec content → inspector display | Screen title, back_behavior, acceptance prose from spec file rendered in inspector rows |

## STRIDE Threat Register

| Threat ID | Category | Component | Disposition | Mitigation Plan |
|-----------|----------|-----------|-------------|-----------------|
| T-05-11 | Tampering | Input value flows to store.apply without validation | mitigate | store.apply validates via argsSchema.safeParse (T-04-01 Phase 4) before reaching command.apply; inspector does no pre-validation |
| T-05-12 | Tampering | Cursor row out of bounds → wrong field activated | mitigate | cursorRow clamped to [0, fields.length-1]; activateField guards with fieldDef.editable check |
| T-05-13 | Spoofing | Diagnostic path substring match false positive (e.g. screenId "s1" matches "s10") | mitigate | Use exact path segment match: diagnostic.path.split("/").includes(screen.id) rather than .includes() substring |
| T-05-14 | Information Disclosure | Full stack trace from store.apply failure shown in inspector | mitigate | Inspector renders apply result as diagnostic; never exposes raw Error objects in render output |
  | T-05-15 | Denial of Service | IME cursor at (0,0) due to missing Focusable implementation | mitigate | Focusable interface implemented; focus propagated to editInput.focused; Pitfall 3 documented in RESEARCH.md |
</threat_model>
