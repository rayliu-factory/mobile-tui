## What Was Built

### IN-05: Never-throw runMigrations contract

`runMigrations` now returns `{ result: unknown; diagnostic: Diagnostic | null }` on every code path:

- **Same-version path** (`fromVersion === toVersion`): returns `{ result: spec, diagnostic: null }` — no-op, original reference preserved
- **Successful migration path**: returns `{ result: current, diagnostic: null }` — migrated spec
- **Unknown version path** (no step in MIGRATIONS chain): returns `{ result: spec, diagnostic: { code: "SPEC_UNSUPPORTED_VERSION", severity: "error", ... } }` — original spec returned unchanged, never throws

The `Diagnostic` type is imported from `../primitives/diagnostic.ts`. The function signature changed from `: unknown` to `: { result: unknown; diagnostic: Diagnostic | null }`.

### parse.ts call site updated

Removed the `try/catch` wrapper around the `runMigrations` call. The new pattern:

```typescript
const { result: migrated, diagnostic: migrationDiag } = runMigrations(
  partition.knownSubset,
  fromVersion,
  toVersion,
);
if (migrationDiag) {
  diagnostics.push({ ...migrationDiag, severity: "warning", path: "" as JsonPointer });
} else {
  partition = { ...partition, knownSubset: migrated as Record<string, unknown> };
}
```

### Migration tests updated (8 tests, was 7)

All `runMigrations(...)` call sites in `index.test.ts` now destructure `{ result }`. The old `toThrow` test was replaced with two new tests: one verifying `SPEC_UNSUPPORTED_VERSION` diagnostic shape, one verifying the never-throw contract holds.
