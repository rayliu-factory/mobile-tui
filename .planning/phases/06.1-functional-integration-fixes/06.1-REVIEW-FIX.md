### WR-01: `tryAdvance()` fire-and-forget errors silently swallowed

**Files modified:** `src/wizard/panes/form-pane.ts`
**Commit:** ee0d8aa
**Applied fix:** Changed `void this.tryAdvance()` to `void this.tryAdvance().catch((err: unknown) => { this.error = err instanceof Error ? err.message : "Unexpected error."; })` at line 99. Unhandled rejections from the async method are now routed to `this.error` so the render loop surfaces them to the user.

---

### WR-02: Non-null assertion on `parseResult.astHandle` unchecked

**Files modified:** `scripts/canvas.ts`, `scripts/wizard.ts`
**Commit:** abfd3c4
**Applied fix:** Expanded the parse-result guard from `if (!parseResult.spec)` to `if (!parseResult.spec || !parseResult.astHandle)` in both scripts. The `!` assertion on `parseResult.astHandle` was removed from the `createStore` call — TypeScript narrowing now guarantees both values are non-null at that point.

---

### WR-03: `autosave` not disposed on empty-render early return

**Files modified:** `scripts/wizard.ts`, `scripts/canvas.ts`
**Commit:** 032831f
**Applied fix:** Added `autosave.dispose()` immediately before `process.exitCode = 1; return;` in the empty-render guard block in both scripts. This removes the dangling `beforeExit` listener that was left registered on the early-exit path (T-04-06 pattern).

---

### WR-04: DataStep and ScreensStep silently drop pending input on Tab

**Files modified:** `src/wizard/steps/data.ts`, `src/wizard/steps/screens.ts`
**Commit:** 282f3f9
**Applied fix:** Added auto-commit logic at the top of the Tab branch in both `handleInput` methods: `const trimmed = this.inputValue.trim(); if (trimmed.length > 0) { this.items.push(trimmed); this.inputValue = ""; }`. For ScreensStep, the auto-commit runs before the min-1 check, so a typed-but-not-entered screen name now satisfies the D-94 requirement instead of triggering a confusing error while the text is still visible.
