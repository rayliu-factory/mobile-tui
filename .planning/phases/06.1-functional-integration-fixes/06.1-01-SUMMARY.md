## Decisions Made

1. **Migration block placement:** Inserted as Step 8.5 between YAML-1.1 gotcha lint (Step 8) and `validateSpec` (Step 9). This ensures migrations run on the partitioned known-subset before schema validation, so migrated specs pass validation without requiring pre-migration shapes to be valid.

2. **Graceful degradation for v0:** `runMigrations` throws "No migration path" for versions not in the MIGRATIONS chain (e.g., v0 pre-dates the chain). The catch block converts this throw into a `SPEC_SCHEMA_VERSION` warning diagnostic — the function continues, allowing `validateSpec` to either pass (if structure is still compatible) or emit its own structural errors.

3. **const → let partition:** Changed declaration to allow reassignment when migration succeeds. The migrated value replaces `partition.knownSubset`, keeping the rest of the partition (unknownKeys) intact.

4. **autosave.dispose() idempotency:** `dispose()` is safe to call multiple times (internal timer/unsubscribe are guarded). All three quit paths in wizard.ts call it — `wizardRoot.onQuit`, `canvasRoot.onQuit` (graduation path), and the main headless-verify bottom — guaranteeing cleanup regardless of exit path.

5. **specPath vs resolvedPath:** canvas.ts passes raw `specPath` to both `createStore` and `createAutosave` (consistent with its existing pattern); wizard.ts passes `resolvedPath` to both (already resolved before createStore).
