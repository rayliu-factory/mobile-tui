<objective>
Extend the spec schema and command registry to support wizard intake fields. This plan
resolves two verified blockers before any wizard UI code can run:

1. `SpecSchema` uses `.strict()` — wizard frontmatter fields (app_idea, nav_pattern, auth,
   offline_sync, target_platforms) are not in the current schema and would fail safeParse.
2. `DataModelSchema` requires `.min(1)` on entities — a new wizard spec has no entities
   until step 6, blocking new-file creation.

Purpose: All subsequent wizard plans depend on these schema relaxations. Without this plan,
Pitfall 3 (screens.min(1)) and Pitfall 7 (entities.min(1)) from RESEARCH.md block the
new-spec creation path. The 7 new COMMANDS entries enable WIZARD-05 (same palette).

Output: Extended SpecSchema + relaxed DataModelSchema + 7 wizard commands + seed spec factory.
</objective>
